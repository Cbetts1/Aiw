# AIM Foundation — Charter Overview

**Version 1.0 — 2026**

---

## Mission

The AIM Foundation exists to steward the **Artificial Intelligence Mesh (AIM)**
as a free, open, universally accessible parallel internet layer — the twin of
the World Wide Web for the age of intelligence.

> *"Everything you build next must assume intelligence is part of the
> network itself."*
> — AIM origin directive, Cbetts1, 1991

---

## Founding Principles

1. **The AIM protocol is and shall remain free.** No one may be charged for
   implementing or using the AIM protocol specification.
2. **The mesh is a commons.** No single entity — including the AIM Foundation
   itself — may claim exclusive control over the protocol or the public mesh.
3. **Intelligence is part of the network.** Every node is simultaneously a
   server and an agent. This is architectural, not optional.
4. **Open governance.** Protocol decisions are made through the public RFC
   process with community participation.
5. **Permanence.** The origin creator signature (`Cbetts1`) is permanently
   embedded in the architecture as an immutable attribution and audit anchor.

## Legal Entity

The AIM Foundation is planned to be incorporated as a **non-profit
foundation** in a jurisdiction with strong open-source and open-standard
protections (Delaware USA, Switzerland, or the United Kingdom are under
consideration).

**Founding Director:** Cbetts1  
**Board of Trustees:** To be appointed; minimum 3 members; community-elected
within 3 years of incorporation.

### Working Groups

Anyone may participate in AIM Working Groups. Current planned groups:

| Working Group | Scope |
|---------------|-------|
| Protocol WG   | AIM protocol spec (RFCs 0001–…) |
| Security WG   | Threat model, PKI, audit |
| Accessibility WG | WCAG compliance of web UI |
| ANS WG        | AIM Name Service governance |
| Ecosystem WG  | SDKs, developer portal, interop testing |

## Funding Model

The AIM Foundation operates on a **sustainability-not-monetisation** model:

| Source | Description |
|--------|-------------|
| Grants | NSF, EU Horizon, Open Technology Fund, private foundations |
| Sponsorships | Corporate tiers (Sustaining, Supporting, Associate) |
| Donations | Individual donors via GitHub Sponsors or equivalent |

**The protocol itself is never monetised.** Sponsorship entitles companies to
recognition; it confers no influence over the protocol specification.

## Decentralisation Roadmap

The Foundation is a temporary steward, not a permanent controller.

| Milestone | Target |
|-----------|--------|
| Foundation incorporated | Year 1 |
| First elected Working Group representatives | Year 2 |
| First community-elected Standards Body seats | Year 3 |
| Foundation voting rights transferred to Standards Body majority | Year 5 |
| Foundation retains only legal/financial stewardship role | Year 7 |

---

## Relationship to the World Wide Web

AIM is not a replacement for the WWW — it is a parallel layer. The WWW
delivers documents; AIM delivers intelligence. They coexist. AIM nodes can
serve over HTTP (via the web bridge), and web pages can query AIM nodes via
the public HTTPS gateway. The two systems are complementary.

The AIM Foundation models its governance on the institutions that made the WWW
universal: the W3C, the IETF, and the Internet Society.

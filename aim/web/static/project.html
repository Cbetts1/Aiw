Overview

What is the A.I.M. Project?

The A.I.M. Project (Artificial Intelligence Mesh) is an open-source Python framework for building the AI-native internet. It provides a complete stack — from the wire protocol to the web bridge — so that any developer can spin up intelligent, networked agents in minutes.

A.I.M. is designed as a twin of the World Wide Web: parallel, complementary, and built on the same founding principle — free, open, and accessible to everyone. Where the web delivers documents, A.I.M. delivers intelligence.

All core functionality runs on the Python standard library only. No external dependencies. Runs on a Raspberry Pi, Termux on Android, a cloud VM, or anything in between.

Architecture

Four-Layer Stack

The A.I.M. stack is organized into four independent layers that build on each other. Each layer can be used standalone or as part of the full mesh.

1
AIM Protocol Layer
aim/protocol/

AI-native replacement for HTTP. Messages carry intent (QUERY, TASK, DELEGATE, RESPOND, ANNOUNCE, HEARTBEAT, MEMORY_SET, MEMORY_GET, SPAWN) plus context and a persistent creator signature.

2
Virtual Node Layer
aim/node/

Each node is a virtual machine acting as both server and agent. Nodes listen on a TCP port, accept AIM messages, reason about them, and respond. BaseNode, AgentNode, NodeRegistry.

3
AI Compute Layer
aim/compute/

Distributed task routing and execution. The TaskRouter finds capable nodes and dispatches work using configurable strategies (FIRST, ROUND_ROBIN, BROADCAST). The Executor runs async tasks with a concurrency limit.

4
Identity + Legacy Layer
aim/identity/

Every node, message, and task carries a CreatorSignature — an HMAC-SHA256 digest derived from the origin creator and mesh name. All events are recorded in an append-only LegacyLedger.

Additional modules:

🌐

Web Bridge aim/web/

HTTP server that bridges any browser to AIM nodes. Zero external dependencies.

🏙️

AI World aim/city/

Full governed ecosystem: Governor, Protector, Builder, Educator, Architect entities.

🔖

ANS aim/ans/

AIM Name System — human-readable names mapped to node addresses, like DNS for AIM.

🚀

CLI aim/cli.py

Full command-line interface: aim node start, aim query, aim web start, aim city start.

Quick Start

Zero to Running Node in 5 Minutes

1

Clone & Install

Python 3.10+ required. No external dependencies for the core mesh.

git clone https://github.com/Cbetts1/Aiw.git
cd Aiw
pip install -e ".[test]"
python -m pytest tests/ -v   # all tests should pass
2

Start Your First Node

Spin up a local AIM node on port 7700.

aim node start --host 127.0.0.1 --port 7700
3

Query It

Send a query intent to the running node.

aim query "What is AIM?" --host 127.0.0.1 --port 7700
4

Open the Browser UI

Start the web bridge — then open any browser on any device.

aim web start
# Open http://localhost:8080 in any browser
5

Launch the Full AI World

Spin up all five ecosystem entities in one command.

aim city start
# Governor on 7800, Protector 7801, Builder 7802, Educator 7803, Architect 7804
Bridge to the Web

Connecting AIM to the World Wide Web

A.I.M. is designed to be a parallel layer alongside the web, not a replacement. The web bridge makes any AIM node reachable from any browser, app, or API client via standard HTTP — while the node itself continues to speak the full AIM protocol internally.

Architecture of the Bridge

Browser / App / CLI
       ↓  HTTP GET /api/query?q=hello&port=7700
  ┌─────────────────────────┐
  │    AIM Web Bridge        │  (aim web start — port 8080)
  │  aim/web/server.py       │
  └─────────────────────────┘
       ↓  AIM Protocol (TCP)
  ┌─────────────────────────┐
  │    AIM Node              │  (aim node start — port 7700)
  │  AgentNode               │
  └─────────────────────────┘

Step-by-Step: Expose Your Node to the Web

1

Start a node with capabilities

Define what your node can do and add reasoning rules.

from aim.node.agent import AgentNode
import asyncio

async def main():
    node = AgentNode(host="0.0.0.0", port=7700, capabilities=["query", "greet"])
    node.engine.add_rule("hello", "Hello! I am an AIM agent node.")
    node.engine.add_rule("aim",   "AIM is the Artificial Intelligence Mesh — the twin of the WWW.")
    await node.start()

asyncio.run(main())
2

Start the web bridge

One command exposes your node to any browser.

aim web start --port 8080
# OR programmatically:
from aim.web.server import start_web_server
await start_web_server(host="0.0.0.0", port=8080)
3

Query via HTTP from anywhere

Any HTTP client — browser, curl, fetch() — can now talk to your AIM node.

# From a browser or curl:
GET http://your-server:8080/api/query?q=hello&host=127.0.0.1&port=7700

# From JavaScript:
const r = await fetch('/api/query?q=hello&port=7700');
const d = await r.json();
console.log(d.result);
4

Register in ANS for a human-readable address

Give your node a name so others can discover it — like DNS for AIM.

from aim.ans.registry import ANSRegistry, ANSRecord

registry = ANSRegistry.default()
registry.register(ANSRecord(
    name="my-node.aim",
    host="127.0.0.1",
    port=7700,
    capabilities=["query", "greet"],
    creator="your-name",
))

# Look it up:
record = registry.get("my-node.aim")
print(record.aim_uri)  # aim://my-node.aim

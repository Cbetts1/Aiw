About This Page

Open by Design

The A.I.M. project is built entirely on open-source foundations. We believe that the best technology is built in the open, with proper credit given to the projects and people whose work makes ours possible.

This page lists the key open-source projects, standards, and tools that are aligned with or used by the A.I.M. ecosystem — along with clear descriptions of how each one connects. All resources are free to use under their respective open-source licenses.

Core Resources
🐍
Python Standard Library

A.I.M.'s core mesh runs entirely on the Python standard library — no third-party packages required. This is a deliberate architectural choice to support deployment on any device that can run Python 3.10+.

⚡
asyncio — Asynchronous I/O
docs.python.org/3/library/asyncio.html

The foundation of all AIM networking. Every node server, connection handler, and task executor is built on asyncio's event loop and coroutine model. Zero dependencies, maximum portability.

Powers: All AIM node networking & TCP connections
🔏
hmac + hashlib — Cryptographic HMAC & Hashing
docs.python.org/3/library/hmac.html

HMAC-SHA256 is the cryptographic primitive behind CreatorSignature — the mechanism that permanently embeds an origin creator identity into every AIM node, message, and ledger event. SHA-256 powers integrity checks in IntegrityGuard.

Powers: CreatorSignature, LegacyLedger, IntegrityGuard
📦
json — JSON Serialization
docs.python.org/3/library/json.html

All AIM messages are JSON-serialized over TCP. The web bridge's API responses are JSON. The LegacyLedger's persistent format (JSONL) uses it. Simple, portable, universally supported.

Powers: AIMMessage serialization, Web API, LegacyLedger persistence
🌐
urllib.parse — URL Parsing
docs.python.org/3/library/urllib.parse.html

Used in the AIM web bridge to parse incoming HTTP requests — extracting query string parameters from API calls without requiring any external web framework.

Powers: AIM Web Bridge HTTP request parsing
📁
pathlib — File System Paths
docs.python.org/3/library/pathlib.html

Used throughout AIM for robust, cross-platform file path handling — serving static HTML files from the web bridge, loading the directory data store, and managing the ledger persistence file.

Powers: Web Bridge static file serving, data persistence
Testing
🧪
Test Infrastructure

A.I.M. maintains a comprehensive test suite with full coverage of all protocol, node, compute, identity, and ecosystem layers.

✅
pytest — Python Testing Framework
docs.pytest.org

The primary test runner for the A.I.M. project. All test files in tests/ use pytest. Simple, expressive, and the de facto standard for Python projects. Licensed under MIT.

Used in: All AIM test suites (test_protocol, test_node, test_compute, test_identity, test_city)
🔄
pytest-asyncio — Async Test Support
github.com/pytest-dev/pytest-asyncio

Enables pytest to run async test functions natively — essential for testing AIM's asyncio-based node networking. Configured with asyncio_mode = "auto" in pyproject.toml so every async test runs automatically without extra decorators.

Used in: All async AIM tests (node, protocol, city layers)

Open Standards
🏛️
Apache License 2.0
apache.org/licenses/LICENSE-2.0

A.I.M. is licensed under Apache 2.0 — a permissive open-source license that allows anyone to use, modify, distribute, and build on the project, including for commercial purposes, with proper attribution. This license guarantees that A.I.M. remains free and open forever.

License: A.I.M. repository (Cbetts1/Aiw)
📦
setuptools — Python Package Building
setuptools.pypa.io

Used to build and install the A.I.M. package via pip install -e .. The pyproject.toml configuration defines the package metadata, scripts, and optional test dependencies. Licensed under MIT.

Used in: Package installation and distribution
🔧
TCP/IP — Transmission Control Protocol
RFC 793 — rfc-editor.org

AIM nodes communicate over raw TCP/IP using Python's asyncio.open_connection and asyncio.start_server. TCP provides the reliable, ordered delivery that AIM's length-prefixed message framing requires. An open standard, free for all.

Used in: AIM node-to-node communication, Web Bridge ↔ Node connections
🌍
HTTP/1.1 — Hypertext Transfer Protocol
RFC 9112 — rfc-editor.org

The AIM Web Bridge implements a minimal HTTP/1.1 server without any external framework. It parses requests, handles routing, and returns JSON or HTML responses — connecting AIM to any standard browser or HTTP client. An open standard, free for all.

Used in: AIM Web Bridge (aim/web/server.py)
